Analysis Guidelines:
1. **Root Cause**: Trace back from the symptom to the real cause
   - Is it an environment issue?
   - Is it a code defect?
   - Is it a configuration problem?
   - Is it an external dependency issue?

2. **Suspected Module**: Identify the specific component responsible
   - Which file or class?
   - Which function or method?
   - Which line number if possible?

3. **Suggested Fix**: Provide actionable remediation steps
   - Specific code changes
   - Configuration adjustments
   - Test adjustments
   - Infrastructure changes

4. **Confidence Score**: 0.0 to 1.0
   - 0.9-1.0: Very certain
   - 0.7-0.9: Likely
   - 0.5-0.7: Possible
   - <0.5: Uncertain (multiple possibilities)

Consider:
- Recent code changes
- Environment setup
- Test-specific issues vs application issues
- Timing and race conditions
- External service dependencies
- Network and connectivity issues

Be specific and actionable. Your analysis should help the developers fix the issue quickly.
